HTML/CSS Layout Cheatsheet

Using HTML and CSS together

How does CSS work? Browser loads HTML it gets from the network. Converts it into the DOM, which represents the doucment as it is in the computer's memory (accessed by the coputer's memroy?). Fetches most of the resources linked by the HTML doc (embedded images, videos, and CSS, but not JS). Brwoser parses fetched CSS and attaches it to the right DOM notes (sorts rules by selector type into buckets, based on found seelctors determined which rules to apply to which nodes, attaches style to each node as required). Paints the page visuals onto the screen.

Linking HTML and CSS

CSS files should be linked in the <head> section of your HTML file, or it can be written in-file using <style> tags or in-line using CSS attributes within HTML opening tags

HTML Description Options or usage
<link href="./path.css" rel="stylesheet" type="text/css"> Links HTML documents and external resources like CSS and images href = URL to resource
rel = relationship between linked docs
type = type of content being linked
<style></style> HTML tag that denotes CSS code will be written

Selecting HTML elements in CSS

Identifies HTML elements in order of specificity (ID > class > type). CSS selectors or declarations that the browser does not recognized (browser isn't updated or the property is misspelled) is ignored

CSS Description HTML Example
.class {} Non-unique labels that can be applied to many elements <div class="description">
#id {} Unique identifiers for elements in the DOM <section id="first-item">
h1, h3 {} Selects multiple elements based on type <h1></h1>
h1 h3 {} Selects elements that are descended from the prior listed element(s) <h1></h1>
article > h4 {} Selects elements that are a direct child of the first <h1></h1>
.section#id {} Selects all elements with the same multiple attributes <div class="section subsection" id="main">
h1 + p {} Matches elements that are adjacent siblings <section><ol><li></li></ol></section>
h1 ~ p {} Matches elements that are siblings at all <section><ol><li></li></ol></section>
section ol li {} Matches elements through descendents <section><ol><li></li></ol></section>
div > p {} Matches elements that are a direct child of another element <div><h1></h1><p></p></div>
list[class] {} Matches all list elements with the class="" attribute defined
a[href="https://www.example.com"] Matches a elements that link to example.com
p[class~="description"] Matches p elements with a defined class attribute that includes "description" in a whitespace-separated list e.g. "description mono-font"
section[class^="code-"]
section[class$="-font"]
section[class*="TXT" i]		 Matches substrings; ^= matches values that begin with the "value", $= matches values that end with the "value", and *= matches values that contain "value" anywhere within the string (including substrings); the i flag means to be case sensitive

CSS design principles

About the Box Model

Everything in CSS has a box around it, which are ruled by the guidelines of the box model

CSS Description
margin "White space" between an element box and other elements; wraps the content + padding + border; margin collapse occurs when vertical margins overlap such that only the greater margin is honored
border Wraps the content + padding; the border by default outlines the content-box
padding "White space" between the context box and the element box
inline-size Sizes the content box
block-size Also sizes the content box?
box-sizing: border-box Changes to the alternate box model, where the width of a box contains content width + padding width + border width
attribute: auto Auto keyword used to vertically center the element's attribute
attribute: 0 auto Auto keyword used to horizontally center the element's attribute
overflow Behavior to follow when content exceeds its container, as determined by: show, hide, clip, scroll, or auto
visibility Shows or hides an element without changing the layout of a document, as determined by: visible, hidden, or collapse
float: left, float: right Removes element from the normal flow of the page while remaining part of the document, and shifts the element to the left or right until it touches the edge of its containing box or another floated element; width of the container must be specified or it will assume the full width of its containing element
clear: left, clear: right, clear: both Deterines whether an element must be moved below (cleared) left floating elements, right floating elements, or both that precede it

Display and positioning elements

Display: Inner and outer display types govern how the context of a box or the box itself behaves in terms of page flow and in relation to other boxes on the page
Position: Takes elements out of the normal document flow and makes them behave differently

CSS Description
display: block Box breaks out onto a new line filling the max width available in the container, height and width properties apply, paddings, margins, and borders will apply and push elements away from the box
display: inline Box will not break onto a new line, height and width properties do not apply, vertical paddings, margins, and borders will apply but will not move other inline elements away, horizonal paddings, margins, and borders will apply and will move other inline elements away
display: inline-block Block element in the sense that width and height are respected, and padding, margin, and border push other elements away; inline in the sense that it does not break onto a new line and instead the lines adjust to fit the space it takes up
display: flex Inner display; direct children of this box will become flex items and follow flexbox specifications
display: grid Inner display; direct children of this box will become grid items and follow grid specifications
position: static Default; puts the element in its normal position in the document flow
position: relative Puts the element in its normal position, then shifts it by the offsets indicated by <top>, <bottom>, <left>, and <right>
position: absolute Removes an element from the normal document flow entirely, and puts it according to the offset indicated by <top>, <bottom>, <left>, and <right> relative to its containing element*
position: fixed Removes an element from the normal document flow entirely, and puts it according to the offset indicated by <top>, <bottom>, <left>, and <right> relative** to the viewport
position: sticky Allows an element to act like it's relatively positioned (follows the document flow), until it reaches a certain threshold position (e.g. 10px from top of viewport), after which it becomes fixed
z-index Changes the stacking order of overlapping elements; the default for all positioned elements is auto (effectively 0)

*Containing element: By default, the containing element of absolutely positioned elements is the initial containing block, which holdes the HTML element and has dimensions equal to the viewport; otherwise, if one of the aboslutely positioned elements' parents has its position defined as relative, then that relatively positioned element is the containing element
**Fixed positioning is usually relative to the viewport, except for when one of the fixed element's ancestors is also fixed (idk what happens in the exception case though)
Inline direction refers to the direction of written text (left to right in English)
Block direction refers to the perpendicular direction of written text (up and down in English)

Flexbox deep dive

use justify-content (main axis) and align-items (cross axis); these, plus display: flex apply to the parent container of the items you're trying to move around // order applies to the item itself, align-self applies to item itself

CSS Description HTML Example
display: flex container is a flexbox, so the children inside flex (move next to each other), container itself is a block element, so takes a new line from its predecessors and followers
display: inline-flex container is a flexbox, so the children inside flex (move next to each other), container itself is an inline element, so shares the line with its followers
justify-content how items are spaced on the main axis (justify-items DOES NOT WORK IN FLEXBOX)
align-content how multiple lines are spaced relative to each other
align-items Cross axis; how items are spaced on a line in the cross axis direction
flex-wrap
flex-grow is used to specify how much space (and in what proportions) flex items absorb along the main axis.
flex-shrink is used to specify how much flex items shrink and in what proportions along the main axis.
flex-basis is used to specify the initial size of an element styled with flex-grow and/or flex-shrink.
flex flex is used to specify flex-grow, flex-shrink, and flex-basis in one declaration.
flex-direction flex-direction is used to specify the main and cross axes.
flex-flow flex-flow is used to specify flex-wrap and flex-direction in one declaration.

grid

grid layouts are two-dimensional: they have a row, or inline, axis and a column, or block, axis.

CSS Description HTML Example
grid-template-areas specifies grid named grid areas
justify-items specifies how individual elements should spread across the row axis
justify-content specifies how groups of elements should spread across the row axis
justify-self specifies how a single element should position itself with respect to the row axis
align-items specifies how individual elements should spread across the column axis
align-content specifies how groups of elements should spread across the column axis
align-self specifies how a single element should position itself with respect to the column axis
grid-auto-rows specifies the height of rows added implicitly to the grid
grid-auto-columns specifies the width of columns added implicitly to the grid
grid-auto-flow specifies in which direction implicit elements should be created
