HTML/CSS Layout Cheatsheet

Using HTML and CSS together

Linking HTML and CSS

CSS files should be linked in the <head> section of your HTML file, or it can be written in-file using <style> tags or in-line using CSS attributes within HTML opening tags

HTML Description Options or usage
<link href="./path.css" rel="stylesheet" type="text/css"> Links HTML documents and external resources like CSS and images href = URL to resource
rel = relationship between linked docs
type = type of content being linked
<style></style> HTML tag that denotes CSS code will be written

Text formatting in CSS

My most commonly used CSS attributes when formatting text on a page

HTML tag Description Example
letter-spacing Adjust the spacing between characters <letter-spacing: 3px;>
text-align Adjust the alignment of text <text-align: center;>
text-transform Transform text to: capitalize, uppercase, lowercase, none, full-width <text-transform: uppercase;>

Selecting HTML elements in CSS

Identifies HTML elements in order of specificity (ID > class > type)

CSS Description HTML Example
.class {} Non-unique labels that can be applied to many elements <div class="description">
#id {} Unique identifiers for elements in the DOM <section id="first-item">
h1, h3 {} Selects elements based on type <h1></h1>
.section#id {} Selects all elements with the named class(es) and ID(s) <div class="section subsection" id="main">
section ol li {} Matches elements along their family lines by searching through descendents <section><ol><li></li></ol></section>

CSS design principles

About the Box Model

Everything in CSS has a box around it, which are ruled by the guidelines of the box model

CSS Description
margin "White space" between an element box and other elements; wraps the content + padding + border; margin collapse occurs when vertical margins overlap such that only the greater margin is honored
border Wraps the content + padding
padding "White space" between the context box and the element box
attribute: auto Auto keyword used to vertically center the element's attribute
attribute: 0 auto Auto keyword used to horizontally center the element's attribute
overflow Behavior to follow when content exceeds its container, as determined by: show, hide, clip, scroll, or auto
visibility Shows or hides an element without changing the layout of a document, as determined by: visible, hidden, or collapse
float: left, float: right Removes element from the normal flow of the page while remaining part of the document, and shifts the element to the left or right until it touches the edge of its containing box or another floated element; width of the container must be specified or it will assume the full width of its containing element
clear: left, clear: right, clear: both Deterines whether an element must be moved below (cleared) left floating elements, right floating elements, or both that precede it

Display and positioning elements

Display: Inner and outer display types govern how the context of a box or the box itself behaves in terms of page flow and in relation to other boxes on the page
Position: Takes elements out of the normal document flow and makes them behave differently

CSS Description
display: block Box breaks out onto a new line filling the max width available in the container, height and width properties apply, paddings, margins, and borders will apply and push elements away from the box
display: inline Box will not break onto a new line, height and width properties do not apply, vertical paddings, margins, and borders will apply but will not move other inline elements away, horizonal paddings, margins, and borders will apply and will move other inline elements away
display: inline-block An element can appear next to another; height and width can be manually adjusted
display: flex Inner display; direct children of this box will become flex items and follow flexbox specifications
display: grid Inner display; direct children of this box will become grid items and follow grid specifications
position: static Default; puts the element in its normal position in the document flow
position: relative Puts the element in its normal position, then shifts it by the offsets indicated by <top>, <bottom>, <left>, and <right>
position: absolute Removes an element from the normal document flow entirely, and puts it according to the offset indicated by <top>, <bottom>, <left>, and <right> relative to its containing element*
position: fixed Removes an element from the normal document flow entirely, and puts it according to the offset indicated by <top>, <bottom>, <left>, and <right> relative** to the viewport
position: sticky Allows an element to act like it's relatively positioned (follows the document flow), until it reaches a certain threshold position (e.g. 10px from top of viewport), after which it becomes fixed
z-index Changes the stacking order of overlapping elements; the default for all positioned elements is auto (effectively 0)

*Containing element: By default, the containing element of absolutely positioned elements is the initial containing block, which holdes the HTML element and has dimensions equal to the viewport; otherwise, if one of the aboslutely positioned elements' parents has its position defined as relative, then that relatively positioned element is the containing element
**Fixed positioning is usually relative to the viewport, except for when one of the fixed element's ancestors is also fixed (idk what happens in the exception case though)
